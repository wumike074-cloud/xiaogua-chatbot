# 设备测试模块 — 设计文档

## 概述

在现有小瓜 chatbot 页面中新增"设备测试"模块，提供分产品的测试向导，测试完成后自动填入结果表格，支持 Excel 导出。

## 布局

页面顶部新增两个标签：**小瓜 Chat** / **设备测试**，默认显示 Chat。

切换至"设备测试"后，页面分为三个区域：

- **左侧**：产品列表（带搜索），宽度约 240px
- **右侧**：测试向导区域，占据剩余空间
- **底部**：测试记录表格，全宽

### 产品列表（左侧）

- 35 个产品，竖向排列，每个产品一个可点击条目
- 顶部搜索框，实时过滤产品名称
- 点击产品名称即开始测试
- 产品名称旁边显示该产品测试步骤数量（如 "X1 · 5步"）
- 无产品步骤数据时不可点击，灰色显示

### 测试向导（右侧）

- 顶部显示产品名称
- 步骤进度条（Step 1/5 → Step 2/5 ...）
- 步骤内容区域，根据步骤类型渲染
- 底部按钮：← 上一步（第一步骤隐藏）、下一步 →（最后步骤显示"完成"）
- 点击完成，数据写入下方表格

### 测试记录表格（底部）

- 所有本次测试过的记录，最新在上
- 永久保留在 localStorage，手动清空
- 字段：SN、产品、订单号、国家、日期、测试人员、固件/软件版本、注册期限、测试结果、备注
- 顶部有"导出 Excel"按钮和"清空记录"按钮
- 每条记录右侧可删除单条

## 步骤类型

三种步骤类型，由产品配置定义：

### 确认型
- 显示说明/检查项文字
- 两个选项按钮：确认 / 不通过
- "确认" → 标记该项通过，进入下一步
- "不通过" → 弹出备注输入框 → 进入下一步（标记为异常）
- 备注文字自动汇总到最终记录

### 输入型
- 显示标题和说明
- 文本输入框
- 可配置默认值（如自动从SN库获取）
- 可配置 placeholder 提示

### 选择型
- 显示标题和问题
- 单选（radio）或多选（checkbox），由配置决定
- 每个选项显示标签文字

## 数据模型

### 产品步骤配置

```js
// 步骤配置文件（初始框架，步骤后续补充）
const testStepsConfig = {
  "X1": [
    { type: "confirm", title: "检查外观", description: "确认设备外壳无划痕、无变形", remarkOnFail: true },
    { type: "input", title: "固件版本", placeholder: "如 R4.2.1", key: "firmware" },
    { type: "select", title: "底座类型", options: ["铝座", "塑座", "无底座"], key: "baseType" },
    { type: "confirm", title: "通电测试", description: "接通电源，确认指示灯正常亮起" },
    { type: "input", title: "注册期限", placeholder: "如 永久 / 2026-12-31", key: "regPeriod" }
  ],
  "X1 lite": [],
  // ... 其余产品待补充
};
```

### 测试记录

```js
{
  sn: "7126R0050",

  country: "巴西",
  date: "2026-05-28",

  result: "pass",       // "pass" | "fail"

    { title: "检查外观", result: "pass", remark: "" },
    { title: "固件版本", result: "pass", value: "R4.2.1" },
    ...
  ],
  time: "14:30:22"
}
```

### localStorage key

- `deviceTestRecords` — 数组，存储所有测试记录
- 不做过期清理，用户手动清空

## 集成点

- **SN 数据库**：测试时输入 SN，自动从 `window.snDB` 查找订单号和国家，预填到表格
- **现有 xlsx 包**：导出 Excel 复用已安装的 `exceljs` 包，支持嵌入二维码（注册码）
- **样式**：复用现有 CSS 变量（`--primary`, `--text`, `--border` 等）
- **字体**：继承现有 `Nunito` 字体

## 导出格式

导出 Excel 包含：
- 每个字段一列
- 每条记录一行
- 如关联到注册码，可选附带二维码列
- 文件名：`设备测试记录_2026-05-28.xlsx`

## 清空逻辑

- "清空记录"按钮二次确认
- 支持导出后再清空
- 提示导出后再清空
